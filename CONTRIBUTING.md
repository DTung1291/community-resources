# Contributing Guide

Thanks for contributing to Web3 Community Toolkit.

## How to Contribute

1. Fork the repository.
2. Create a branch with a clear name.
3. Add or improve one resource per pull request when possible.
4. Open a pull request with clear description and context.

## Content Standards

- Keep templates practical and easy to reuse.
- Use concise headings and checklist-style sections.
- Avoid project-specific private information.
- Use English for shared templates unless a localized version is intended.

## File Naming

- Use lowercase kebab-case file names.
- Place files in the correct folder by topic.
- Prefer one template per file.

## Pull Request Checklist

- The file is in the correct section.
- Wording is clear and actionable.
- No sensitive data, keys, or personal information.
- README links are updated if new files are added.

## Suggested Commit Message Style

- docs: add scam prevention checklist
- docs: improve weekly ambassador report template
- chore: update README links
