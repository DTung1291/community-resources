# User Support Workflow

## Intake
- Ask users to open support ticket using one format:
  - Issue type
  - Wallet or account ID (non-sensitive)
  - Time of incident
  - Screenshots

## Triage Priority
1. Account security or funds risk.
2. Access blockers (role, verification, permissions).
3. General product or community questions.

## Resolution Process
1. Acknowledge ticket within target SLA (for example 15 minutes).
2. Confirm required details and ownership proof where needed.
3. Escalate technical issues to core team with context.
4. Resolve and send clear next steps.
5. Mark ticket closed with outcome summary.

## Quality Standards
- Never request seed phrase or private keys.
- Keep replies concise, respectful, and action-focused.
- Store final resolution notes for future reference.
